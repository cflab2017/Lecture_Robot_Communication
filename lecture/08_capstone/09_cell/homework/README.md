## 문제 1 — {TRIG2} 음수 좌표로 한 사이클 수행
- 파일명: `homework_01/main.py`
- 핵심 개념: 음수 좌표 인코딩, `IR400~411` 디코딩 검증

### 요구사항
- 비전에 `{TRIG2}` 를 보내 음수 좌표 `{-45.10, 210.00, -15.0}` 를 받는다.
- 그 좌표로 픽앤플레이스 한 사이클(픽 접근 → 하강 → 집기 → 상승 → 플레이스 → 놓기 → GO HOME)을 수행한다.
- 픽 하강 직후 로봇 직교 현재값 `IR400~411` 을 되읽어 디코딩하고, X가 정확히 `-45.10` 으로 들어갔는지 `OK`/`FAIL` 로 출력한다.

### 예상 출력
```
=== 과제1: {TRIG2} 음수 좌표 픽앤플레이스 ===
[비전] TRIG2 → X=-45.1 Y=210.0 R=-15.0
[로봇] 픽 접근 (X=-45.1) 완료
...
[로봇] GO HOME 완료
----------------------------------------
픽 위치 IR400~411 디코딩 X..C = [-45.1, 210.0, 5.0, 0.0, 0.0, -15.0]
음수 좌표 왕복: X=-45.1 (기대 -45.1)  OK
```

### 힌트
- `split_word(round(-45.10*1000))` → `(20436, -1)`. `RobotMaster.write_holding` 이 각 word를 `& 0xFFFF` 로 인코딩하므로 음수도 그대로 전달하면 된다.
- 디코딩은 word 2개를 signed 16bit 로 본 뒤 `combine_word(low, high) * 0.001`.
- 로봇 완료는 **2단계 폴링**(먼저 `IR524==2` 시작 → `IR524==1 & IR200==1` 완료), 리셋 후 한 박자 대기, 그리퍼는 `IR769==2 & HR1606==0`.

## 문제 2 — 여러 부품 연속 처리 + 평균 사이클 타임
- 파일명: `homework_02/main.py`
- 핵심 개념: 연결 재사용, 반복 루프, 사이클 타임 측정

### 요구사항
- `--cycles N` 옵션으로 부품 N개를 연속 처리한다(`TRIG`/`TRIG2` 를 번갈아 사용).
- 로봇·그리퍼 연결은 **한 번만** 열어 모든 사이클에 재사용한다(매 사이클 새 연결 금지).
- 각 사이클의 cycle time 을 측정해, 마지막에 평균/최소/최대를 출력한다.

### 예상 출력
```
=== 과제2: 3개 부품 연속 처리 ===
  사이클 1/3 (TRIG) 완료 — cycle time 4.16s
  사이클 2/3 (TRIG2) 완료 — cycle time 4.19s
  사이클 3/3 (TRIG) 완료 — cycle time 4.18s
----------------------------------------
전체 3 사이클 — 평균 4.18s (min 4.16s / max 4.19s)
```
(cycle time 숫자는 환경에 따라 다릅니다.)

### 힌트
- `with RobotMaster(...) as robot, RobotMaster(...) as gripper:` 블록 **안에서** N번 루프를 돈다.
- 반복 처리에서는 **상태 잔상(latch)** 이 특히 잘 드러난다. 로봇 2단계 폴링과 그리퍼 `HR1606==0` 동반 확인이 없으면 두 번째 사이클부터 좌표가 갱신되지 않은 채 통과하거나 명령이 잠긴다.
- `t0 = time.time()` 으로 사이클 시작을 잡고, 끝에서 `time.time() - t0` 를 리스트에 모은다.
