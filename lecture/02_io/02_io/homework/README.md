# 과제 - 02. I/O 통신

## 문제 1 — 3스테이션 연속 핸드셰이크
- 파일명: `homework_01/main.py`
- 핵심 개념: 핸드셰이크 일반화, 데이터/흐름 분리, $DI 응답 초기화

### 요구사항
- 본문 예제의 단일 GRIP/UNGRIP 핸드셰이크를 **3개 스테이션의 연속 시퀀스**로 확장한다.
  (예: ST1 집기 → ST2 놓기 → ST3 집기)
- 각 스테이션은 `(제목, $DO핀, $DO값, $DI응답핀)` 형태의 **데이터(리스트)** 로 두고, 반복문으로 처리한다.
- 각 스테이션마다 `$DO` 출력 → PLC 응답 → `WAIT FOR $DI` 순서를 출력하고, 마지막에 `전체 3스테이션 완료 ✅` 를 출력한다.

### 예상 출력 (일부)
```
3스테이션 연속 I/O 핸드셰이크 시작

[1] == ST1 PICK  (집기) : $DO[6] ON,  $DI[1] 그립완료 대기 ==
    [ROBO] $DO[6]=TRUE 출력
    [PLC ] $DO[6]=ON 감지 → 동작 중...
    [PLC ] 동작 완료 → $DI[1]=TRUE 응답
    [ROBO] WAIT FOR $DI[1]==TRUE ...
    [ROBO] $DI[1]==TRUE 확인 → 다음 동작 진행
...
전체 3스테이션 완료 ✅
```

### 힌트
- 스테이션 정보를 리스트로 두면 추가/삭제가 쉽다. `for title, do, val, di in STATIONS:`
- **다음 `WAIT FOR` 전에 해당 `$DI` 비트를 `False` 로 초기화**해야 즉시 통과하지 않는다. 이게 빠지면 두 번째부터 대기 없이 지나간다.

## 문제 2 — 핸드셰이크 타임아웃 감시
- 파일명: `homework_02/main.py`
- 핵심 개념: 블로킹 대기의 위험, 타임아웃, `time.monotonic()`

### 요구사항
- `WAIT FOR $DI` 가 무한 대기에 빠지지 않도록 **타임아웃(예: 2초)** 을 둔다.
- 응답이 기한 안에 오면 경과 시간과 함께 정상 진행을, 끝내 오지 않으면 `⚠️ 타임아웃!` 경고를 출력한다.
- 정상 케이스(응답 옴)와 고장 케이스(무응답) 두 가지를 모두 보여준다.

### 예상 출력
```
핸드셰이크 타임아웃 감시 데모 시작

== CASE A: 정상 응답 ($DO[6] ON → $DI[1]) ==
   [ROBO] $DO[6]=TRUE 출력
   [ROBO] WAIT FOR $DI[1]==TRUE ... (timeout 2.0s)
   [ROBO] $DI[1]==TRUE 확인 (0.50s) → 진행

== CASE B: 무응답 ($DO[6] OFF → $DI[4]) ==
   [ROBO] $DO[6]=FALSE 출력
   [ROBO] WAIT FOR $DI[4]==TRUE ... (timeout 2.0s)
   [WARN] ⚠️  $DI[4] 응답 없음 — 2.0s 타임아웃! (케이블/PLC 점검 필요)

결과: CASE A=OK, CASE B=TIMEOUT
타임아웃 발생 ❌ — 위 ⚠️ 항목 점검
```

### 힌트
- 경과 시간은 `start = time.monotonic()` 후 `time.monotonic() - start` 로 잰다(시스템 시계 변경에 영향받지 않음).
- 무응답 케이스는 "PLC가 영영 응답하지 않는" 상황이므로, 응답 시점을 `None` 으로 두고 모사한다.
- 타임아웃이 발생하면 그 스텝을 **실패로 처리**하고 멈춘다(현장에서는 계속 진행하면 오동작으로 이어짐). CASE B는 의도된 타임아웃이므로 종료 코드가 1이다.
