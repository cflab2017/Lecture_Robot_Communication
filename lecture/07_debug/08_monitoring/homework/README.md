## 문제 1 — 주어진 hex 프레임 손해독
- 파일명: `homework_01/main.py`
- 핵심 개념: Modbus 프레임 구조 `[Function Code][Data]`, 읽기 요청/응답 해석, byte수=개수×2

### 요구사항
- 다음 두 프레임(16진수)을 사람이 읽게 해독한다.
  1. **요청** `04 01 90 00 02` — Function Code, 시작주소(10진), 개수를 출력.
  2. **응답** `03 06 00 64 00 00 04 4E` — Function Code, byte 수, 레지스터 값 리스트(10진)를 출력.
- 각 값을 손계산 결과와 비교(`assert`)해 맞으면 마지막 줄에 `프레임 해독 완료 ✅`.

### 예상 출력
```
[1] 요청 FC=0x04, 시작주소=400(0x0190), 개수=2
[2] 응답 FC=0x03, byte수=6, 값=[100, 0, 1102]
프레임 해독 완료 ✅
```

### 힌트
- 16bit 값은 상위·하위 바이트를 합친다: `(hi << 8) | lo`. Modbus는 **big-endian**(상위 먼저).
- 응답의 두 번째 바이트는 **byte 수**입니다. Word 읽기는 `byte수 = 개수 × 2` → byte 6은 레지스터 **3개**.
- `0x0190 = 400`, `0x044E = 1102`. 손으로 한 번 계산해 보고 코드와 맞춰 보세요.

## 문제 2 — 값 리스트를 10h 요청 프레임으로 인코딩 (역방향)
- 파일명: `homework_02/main.py`
- 핵심 개념: 손해독의 반대 방향(encode), `10h` Write Multiple Registers 송신 포맷, 왕복 검증

### 요구사항
- 시작주소 `201`, 값 `[111, 222, 333]`을 **10h(Write Multiple Registers) Master 송신 프레임**(16진)으로 만든다.
- 프레임 구조: `[10][시작주소(2)][개수(2)][byte수(1)][값들(개수×2)]`.
- 만든 프레임이 매뉴얼 예제 `10 00 C9 00 03 06 00 6F 00 DE 01 4D`와 같은지 확인하고, **다시 디코드**해 원래 (시작주소, 값)이 복원되는지(왕복) 검증한다.

### 예상 출력
```
입력: 시작주소=201, 값=[111, 222, 333]
10h 요청 프레임: 10 00 C9 00 03 06 00 6F 00 DE 01 4D
재해독: 시작주소=201, 개수=3, 값=[111, 222, 333]
인코딩/왕복 검증 완료 ✅
```

### 힌트
- `byte 수 = 개수 × 2` 칸을 잊지 마세요. **10h 송신**에는 byte수+데이터가 있지만, **응답 에코**(`10 00 C9 00 03`)에는 없습니다.
- 각 값은 2바이트로 분해: 상위 `(v >> 8) & 0xFF`, 하위 `v & 0xFF`. `0x006F=111`, `0x00DE=222`, `0x014D=333`.
- 16bit를 넘는 값이 새지 않도록 `v & 0xFFFF`로 먼저 자릅니다.
