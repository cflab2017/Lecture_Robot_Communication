## 문제 1 — 정수 리스트 패킷 프레이밍/역파싱
- 파일명: `homework_01/main.py`
- 핵심 개념: HIWIN 패킷 포맷(`{ }` 감쌈, `,` 구분), 왕복(round-trip) 검증

### 요구사항
- 정수 리스트를 `{a,b,c}` 형태의 패킷 문자열로 만드는 `frame(values)` 함수를 작성한다.
- 패킷 문자열에서 `{ }`를 벗기고 `,`로 나눠 정수 리스트로 되돌리는 `unframe(packet)` 함수를 작성한다.
- `[123, 456]`, `[90000, -45100, 30]`, `[7]` 세 경우를 프레이밍 → 역파싱해, 원래 리스트와 같은지 `OK`/`FAIL`로 출력한다.

### 예상 출력
```
[123, 456]  -> "{123,456}"  -> [123, 456]  OK
[90000, -45100, 30]  -> "{90000,-45100,30}"  -> [90000, -45100, 30]  OK
[7]  -> "{7}"  -> [7]  OK
```

### 힌트
- 프레이밍: `"{" + ",".join(str(v) for v in values) + "}"`.
- 역파싱: `packet.strip().strip("{}")`로 중괄호를 먼저 벗긴 뒤 `split(",")`, 각 토큰을 `int()` 변환.
- 음수·여러 자리 정수도 같은 규칙으로 왕복됩니다.

## 문제 2 — non-format(ASCII decimal) 인코딩/디코딩
- 파일명: `homework_02/main.py`
- 핵심 개념: non-format(시작/끝·구분 기호 없음), 문자 ↔ ASCII CODE(10진수)

### 요구사항
- 문자열을 ASCII CODE(10진수) 리스트로 바꾸는 `encode_nonformat(text)` 함수를 작성한다. (예: `"AB"` → `[65, 66]`)
- 그 리스트를 다시 문자열로 되돌리는 `decode_nonformat(codes)` 함수를 작성한다.
- `"AB"`, `"123"`, `"OK"` 세 경우를 인코딩 → 디코딩해, 원래 문자열과 같은지 `OK`/`FAIL`로 출력한다.

### 예상 출력
```
"AB"  -> [65, 66]  -> "AB"  OK
"123"  -> [49, 50, 51]  -> "123"  OK
"OK"  -> [79, 75]  -> "OK"  OK
```

### 힌트
- 인코딩은 `ord(문자)`, 디코딩은 `chr(코드)`로 1:1 대응됩니다.
- **주의**: non-format에서 문자열 `"123"`은 정수 `123`이 아니라 문자 `'1','2','3'` = `[49, 50, 51]` 세 바이트입니다.
