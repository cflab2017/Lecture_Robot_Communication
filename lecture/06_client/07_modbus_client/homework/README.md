## 문제 1 — 그리퍼 상태(IR769) 사람이 읽는 문자열로 모니터링
- 파일명: `homework_01/main.py`
- 핵심 개념: Input Register 폴링, 상태 코드 매핑, latch 함정 회피

### 요구사항
- 로봇(Master Client)이 그리퍼의 `IR[769]`(상태)를 읽어 코드(0~3)를 **사람이 읽는 문자열**(`Idle`/`Busy`/`Pos`/`Hold`)로 출력한다.
- 열기 명령(`HR[1600]=1`, `HR[1601]=5000`, `HR[1606]=1`)을 보내고, 완료까지 `IR[769]`를 **폴링**하며 상태가 바뀔 때마다 출력한다.
- 완료 판정은 `IR[769]==2` 만으로는 부족하다. 직전 동작의 상태가 latch 되어 즉시 통과하므로, `HR[1606]==0`(실행 플래그 자동 리셋)을 **함께** 확인한다.
- `argparse --host(127.0.0.1) --port(1503)` 를 지원한다.

### 예상 출력
```
=== 그리퍼 상태 모니터링 (IR[769]) ===
동작 전 상태 IR[769]=0  ->  Idle
열기 명령 전송 -> 상태 폴링 시작
  IR[769]=0  ->  Idle
  IR[769]=1  ->  Busy
  IR[769]=2  ->  Pos  (위치 IR[770]=5000 = 50.00mm)
관찰된 상태 흐름: Idle -> Busy -> Pos
상태 모니터링 완료 ✅
```
> 직전 실습으로 그리퍼가 이미 `Pos`(2) 상태면 첫 줄이 `Idle` 대신 `Pos` 로 나올 수 있습니다(latch). 이것이 바로 `IR[769]==2` 만으로 완료를 판단하면 안 되는 이유입니다.

### 힌트
- `STATUS = {0: "Idle", 1: "Busy", 2: "Pos", 3: "Hold"}` 딕셔너리로 매핑.
- 폴링 조건: `m.read_input(769, 1)[0] == 2 and m.read_holding(1606, 1)[0] == 0`.
- 직전 값과 다를 때만 출력하면 상태 변화만 깔끔하게 찍힙니다.

## 문제 2 — IEEE754 왕복 변환 / SWAP_WORD 검증
- 파일명: `homework_02/main.py`
- 핵심 개념: 실수의 Modbus 전송, IEEE754 인코딩/디코딩, word 순서 보정

### 요구사항
- `_shared/word_tools.py`의 `ieee754_encode` / `ieee754_decode` / `swap_word` 를 불러온다.
- 실수 `10.5` 를 **인코딩(실수→정수)** 한 뒤 다시 **디코딩(정수→실수)** 해, 원래 값과 같은지 `OK`/`FAIL`로 확인한다. (50.0 등 다른 값도 함께)
- `swap_word(10)` 이 `655360`, `swap_word(65535)` 가 `4294901760` 인지 검증한다.
- pymodbus 없이 `word_tools` 만으로 동작한다(서버 불필요).

### 예상 출력
```
=== IEEE754 왕복 변환 (실수 -> 정수 -> 실수) ===
    10.5 -> encode ->   1093140480 -> decode -> 10.5      OK
    50.0 -> encode ->   1112014848 -> decode -> 50        OK
  -12.25 -> encode ->   3242459136 -> decode -> -12.25    OK

=== SWAP_WORD (High/Low word 교환) ===
swap_word(10) = 655360  (기대 655360)  OK
swap_word(65535) = 4294901760  (기대 4294901760)  OK
```

### 힌트
- `enc = ieee754_encode(10.5)` → `1093140480`, `ieee754_decode(enc)` → `10.5`.
- 부동소수점 비교는 `abs(dec - value) < 1e-6` 로.
- `swap_word(v)` 는 `((v & 0xFFFF) << 16) | ((v >> 16) & 0xFFFF)` — 상·하위 word 자리 교환.
