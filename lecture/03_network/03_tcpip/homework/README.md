## 문제 1 — `{TRIG2}` 음수 좌표 처리·표시
- 파일명: `homework_01/main.py`
- 핵심 개념: 패킷 송수신, `{ }` 벗기기, 음수 좌표 파싱

### 요구사항
- 비전 서버에 `{TRIG2}`를 송신한다. (`--host` / `--port` 인자 지원, 기본 `127.0.0.1:6000`)
- 응답 `{-45.10,210.00,-15.0}`을 받아 `{ }`를 벗기고 `,`로 분리한 뒤, 각 값을 `float`으로 변환한다.
- X, Y, 회전각을 부호(`+`/`-`) 정렬로 출력하고, 음수 좌표가 섞였는지 한 줄로 안내한다.

### 예상 출력
```
송신: {TRIG2}
수신: {-45.10,210.00,-15.0}
파싱 결과 -> X=-45.10, Y=+210.00, 회전각=-15.0  (음수 좌표 포함)
```

### 힌트
- `body = data.strip().strip("{}")` 로 중괄호를 먼저 제거해야 첫/마지막 값이 깨지지 않습니다.
- 문자열 비교가 아니라 `float()` 변환 후 `< 0` 으로 음수를 판정합니다.

## 문제 2 — 여러 번 촬영 요청 루프 후 평균/개수 출력
- 파일명: `homework_02/main.py`
- 핵심 개념: 한 연결 위에서 `CWRITE`/`CREAD` 반복, 통계 집계

### 요구사항
- **하나의 소켓 연결**을 열고 `{TRIG}`를 `--count`회(기본 3회) 반복 송수신한다.
- 매 응답을 파싱해 검출/검출 없음(`{0.00,0.00,0.0}`)을 구분 출력한다.
- 마지막에 검출 개수와 검출된 좌표의 평균 X, Y를 출력한다. (검출 없음은 평균에서 제외)

### 예상 출력
```
[1/3] X=123.45, Y=67.89, R=30.0  (검출)
[2/3] X=123.45, Y=67.89, R=30.0  (검출)
[3/3] X=123.45, Y=67.89, R=30.0  (검출)
------------------------------
요청 3회 중 검출 3회
평균 좌표 -> X=123.45, Y=67.89
```

### 힌트
- 매 요청마다 새 소켓을 열지 말고, `with socket(...)` 블록 안에서 `for` 루프로 송수신만 반복합니다.
- 검출 판정: `not (x == 0.0 and y == 0.0 and r == 0.0)`.
